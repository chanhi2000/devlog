<p>Xcode’s jump bar – the popup menu describing all properties and methods in the current class, activated using Ctrl-6 –&nbsp;is a simple way to navigate around your file, particularly when you know you can just type a few letters to automatically filter the list.</p>
<p>However, if you’re finding the jump bar is getting messy, you should try to break it up by adding markers to your code. These come in a few flavors – <code>MARK</code>, <code>TODO</code>, and <code>FIXME</code> – but all help make your jump bar easier to read.</p>
<p>Try adding this special comment before any method:</p>
<pre class=" language-swift"><code class=" language-swift"><span class="token comment">// TODO: This isn’t finished yet</span></code></pre>
<p>In the jump bar that will appear as a blue and white list icon so it sticks out. Now try adding this somewhere:</p>
<pre class=" language-swift"><code class=" language-swift"><span class="token comment">// FIXME: This code is awful.</span></code></pre>
<p>That will show up in the jump bar with an orange bandaid – it really stands out! </p>
<p>The third option is <code>MARK</code>, which renders your text in bold –&nbsp;it’s great for splitting up the jump bar, and is commonly used to mark divisions in your file.. Try this:</p>
<pre class=" language-swift"><code class=" language-swift"><span class="token comment">// MARK: UITableViewDataSource methods</span></code></pre>
<p>All of these markers can have dashes placed before or after the text to make separators. For example:</p>
<pre class=" language-swift"><code class=" language-swift"><span class="token comment">// FIXME: - This code is really broken -</span></code></pre>
<p>That will appear in the jump bar with a dividing line before and after the marker. You can just create the separator if you want, like this:</p>
<pre class=" language-swift"><code class=" language-swift"><span class="token comment">// MARK: -</span></code></pre>
<p>That just creates a dividing line with no accompanying text.</p>
