<p>When you connect your storyboard to your code Xcode inserts two special markers: <code>@IBAction</code> and <code>@IBOutlet</code>. Both of these effectively do nothing when you build your app, but they are still important because Xcode uses them to understand which of your properties and methods are relevant to Interface Builder.</p>
<p>In the case of <code>@IBOutlet</code>, this is a connection from an Interface Builder user interface component –&nbsp;e.g. a <code>UIButton</code> – to a property in a view controller or other piece of Swift code. To the left of the code you should see a black circle with a ring around it, which is Xcode’s visual confirmation that a given <code>@IBOutlet</code> has an active connection.</p>
<p>If you’re using the assistant editor with IB in one pane and your code in the other, you can hover over that circle with your mouse pointer to see the UI component highlighted.</p>
