<p>The <code>override</code> is used when you want to write your own method to replace an existing one in a parent class. It's used commonly when you're working with <code>UIViewControllers</code>, because view controllers already come with lots of methods like <code>viewDidLoad()</code> and <code>viewWillAppear()</code>. When you want to override these default methods, you need to specify this with the <code>override</code> keyword.</p>
<p>Now, you might be wondering why the <code>override</code> keyword is even needed, but it's really about ensuring your code is safe – if you write a method and accidentally name it the same as something that already exists, Xcode will simply refuse to build your app until you add in the <code>override</code> keyword. Similarly, if you use <code>override</code> on a method that doesn't override something that already exists, Xcode will refuse to build.</p>
