<p>Auto Layout constraints a powerful way of expressing your layouts, but when they go wrong they can be hard to debug. Fortunately, all constraints have a built-in <code>identifier</code> property that you can use to identify them uniquely.</p>
<p>It’s an optional string so you don’t have to provide anything, but if you <em>do</em> set an identifier you’ll find it much easier to see where your constraints are going wrong because Xcode will use those identifiers in its debug logs.</p>
<p>If you create your constraints in code, just set the <code>identifier</code> property to a string as you go –&nbsp;“Main Title Horizontal Center” for example. If you use Interface Builder, you can select any constraint and you’ll see a dedicated “Identifier” text property you can fill in.</p>
<p>There is literally no reason not to add identifiers to your constraints –&nbsp;they don’t affect your layouts at all, but they do make layout debugging significantly easier!</p>
